### 一、 HTML：网页的骨架与语义

HTML（超文本标记语言）的核心在于**“语义化”**。在编写代码时，不仅是为了让内容显示出来，更是为了让浏览器、搜索引擎清晰地理解这段代码代表什么。

#### 1. 根骨架结构

* `<!DOCTYPE html>`：文档类型声明。告诉浏览器当前页面使用的是 HTML5 标准解析（扩展知识）。
* 
`<html>`：网页的根元素，所有内容都必须包含在其中 。


* 
`<head>`：网页头部，主要存放给浏览器解析的元数据（如CSS样式、字符集配置等），里面的内容通常不会直接展示给用户 。


* `<meta charset="UTF-8">`：定义网页的字符编码，防止乱码（扩展知识）。
* 
`<title>`：定义网页在浏览器标签页上显示的标题 。




* 
`<body>`：网页主体，包含了所有呈现给用户看的可见内容，如文字、图片、视频等 。



#### 2. 文本与排版标签

* 
**标题级标签 (`<h1>` - `<h6>`)**：用于定义各级标题 。`<h1>` 通常代表页面主标题，对SEO（搜索引擎优化）非常重要，一个页面建议只用一次（扩展知识）。


* 
**段落标签 (`<p>`)**：用于定义文本段落，浏览器会在段落前后自动添加空白边距 。


* **文本格式化标签**：
* 
`<b>` / `<strong>`：加粗文字。其中 `<strong>` 具有强烈的强调语义，适合标记重要内容 。


* 
`<i>` / `<em>`：倾斜文字。`<em>` 同样具有强调语义 。


* 
`<s>` / `<del>`：添加删除线。`<del>` 表示文档中已被删除的文本，常用于电商打折前的原价展示 。


* 
`<u>` / `<ins>`：添加下划线。`<ins>` 表示文档中新插入的文本 。





#### 3. 布局与无语义标签

* 
**`<div>`**：块级元素（Block-level）。它**独占一行**，宽度默认撑满父元素，高度由内容撑开，且**可以自由设置宽高** 。常作为大容器用于划分页面区块。


* 
**`<span>`**：行内元素（Inline）。它**不独占一行**（一行可以显示多个），宽高由内部内容撑开，**无法直接设置宽高** 。常用来包裹段落中的特定文字以单独设置样式。


* **语义化布局标签**（扩展补充）：如 `<header>`（页眉）、`<nav>`（导航栏）、`<main>`（主要内容）、`<footer>`（页脚）。它们的展示效果和 `<div>` 一样，但能极大提升代码的可读性。

#### 4. 媒体与超链接

* 
**`<a>` (Anchor)**：超链接标签，用于页面跳转 。


* 
`href`：指定目标资源的URL路径 。


* 
`target`：指定打开方式。`_self` 为当前页打开，`_blank` 为在新标签页中打开空白页面 。




* 
**`<img>`**：图片标签，属于单标签 。


* 核心属性：`src`（图片路径）、`width` 和 `height`（控制尺寸）、`alt`（图片加载失败时的替换文本，扩展知识） 。




* 
**`<video>`**：视频标签 。


* 核心属性：`src`（视频路径）、`controls`（显示播放/暂停等控件）、`width` / `height` 。





#### 5. 表单与交互 (`<form>`)

表单是网页采集用户数据（如登录、搜索、注册）的核心机制 。

* 
**`<form>`**：表单整体容器 。


* 
`action`：规定数据提交的目标URL地址 。


* 
`method`：提交方式。`GET`（参数拼接在URL后，大小受限且不安全），`POST`（参数放在请求体中，无大小限制，相对更安全） 。




* 
**`<input>`**：最核心的表单项，通过 `type` 属性变幻出各种输入框（如 `text` 文本框、`password` 密码框、`radio` 单选框、`checkbox` 复选框） 。


* 
**`<button>`**：按钮，通常分为 `submit`（提交表单）、`reset`（重置表单）和普通的 `button` 。


* 
**`<select>` 与 `<textarea>**`：`<select>` 用于定义下拉选择列表，`<textarea>` 用于定义允许多行输入的文本域 。



#### 6. 表格 (`<table>`)

主要用于规整地展示结构化数据 。

* 
`<table>`：表格的最外层包裹标签 。


* 
`<thead>` 与 `<tbody>`：分别定义表格的头部区域和主体数据区域，使结构更清晰 。


* 
`<tr>` (Table Row)：定义表格中的一行 。


* 
`<td>` (Table Data)：定义普通的单元格 。


* 
`<th>` (Table Header)：定义表头单元格，文字通常会默认加粗并居中 。

### 二、 CSS：网页的皮肤与排版

如果HTML是骨架，那CSS（层叠样式表）就是决定网页颜值的皮肤和决定元素排布的规则 。

#### 1. CSS 引入方式

* 
**行内样式**：直接在标签内部写 `style="..."`。耦合度极高，仅在配合 JavaScript 动态修改特定元素样式时使用 。


* 
**内部样式**：写在 `<head>` 中的 `<style>` 标签里。适合单页面且样式较少的情况 。


* 
**外部样式**：将CSS代码提取到独立的 `.css` 文件中，利用 `<link rel="stylesheet" href="...">` 引入。这是工程化开发中最推荐的解耦方式 。



#### 2. CSS 选择器进阶

选择器决定了“你的样式要作用在谁身上” 。优先级顺序为：**ID选择器 > 类选择器 > 元素选择器** 。

* **基础选择器**：
* 
**元素选择器**：直接写标签名（如 `h1 {...}`），选中页面所有该标签 。


* 
**类选择器**：以 `.` 开头（如 `.cls {...}`），选中带有指定 `class` 属性的元素，复用性极高 。


* 
**ID 选择器**：以 `#` 开头（如 `#hid {...}`），选中唯一指定的 ID 元素（HTML中ID应该保持唯一） 。




* **高级选择器**：
* 
**分组选择器**：用逗号分隔（如 `h1, h2 {...}`），可以同时给多个不同的选择器应用同一套样式 。


* 
**属性选择器**：根据元素的属性甚至属性值来精确选择。例如 `input[type="text"]` 只选中输入框类型为 text 的元素 。


* 
**后代选择器**：用空格分隔（如 `form input {...}`），选中 `form` 内部嵌套的所有 `input` 后代元素 。





#### 3. 核心布局模型：盒子模型 (Box Model)

在网页中，一切可见的元素皆可看作是一个矩形的“盒子”。真正理解页面排版的关键，就是理解盒子的四个层级 ：

1. **Content (内容)**：元素的实际内容区域（文字或图片），由 `width` 和 `height` 控制尺寸。
2. 
**Padding (内边距)**：内容与边框之间的距离。设置 `padding: 20px` 会让内容往内部缩进 。


3. 
**Border (边框)**：包围在内边距和内容外围的线条，可以设置粗细、样式（实线 solid、虚线 dashed 等）和颜色 。


4. 
**Margin (外边距)**：盒子与其他盒子之间的距离，决定了元素在页面上的相对位置。常利用 `margin: 0 auto;` 配合定宽来实现元素的**水平居中**显示 。



* 
提示：上述三个间距属性都可以按 `上 右 下 左` 的顺时针顺序单独设置（如 `padding-top`） 。



#### 4. 现代排版利器：Flex 弹性布局

Flex（Flexible Box）是一种一维的布局模型，专门为了解决元素在容器内的排列、对齐和分配空间而生，完美解决了传统浮动布局的痛点 。

* 
**开启弹性盒**：给父容器设置 `display: flex;`，此时它的直接子元素就变成了弹性项目 (Flex item) 。


* **控制主轴方向 (`flex-direction`)**：
* 
`row`（默认）：子元素水平向右排成一行 。


* 
`column`：子元素垂直向下排成一列 。




* **控制主轴上的对齐方式 (`justify-content`)**：
* 
`flex-start`：所有元素挤在起始端 。


* 
`center`：所有元素紧凑地居中对齐 。


* 
`space-between`：两端元素贴边，中间的元素平分剩余间距 。


* 
`space-around`：每个元素两侧的间距相等（平分剩余空间） 。




* 
**控制交叉轴上的对齐 (`align-items`)**：常配合 `align-items: center;` 实现子元素在垂直方向上的完美居中对齐 。
